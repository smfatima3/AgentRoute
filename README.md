# AgentRoute

A deterministic, training-free communication layer for multi-agent LLM systems.

AgentRoute routes each query to a single specialist agent using a pattern-based
domain classifier, an LRU result cache, and an optional small-model fallback for
low-confidence queries. It issues exactly one LLM call to the chosen specialist
per resolved query, uses no learned components, and requires no training. The
design adapts the Actor Architecture of Jang et al. (2005) to the LLM setting.

> **Status: work in progress.** This repository accompanies a paper draft under
> preparation. The code is functional and validated, but the full experimental
> runs are still ongoing — reported numbers, tables, and figures are not yet
> final. See [Experiment status](#experiment-status) below.

## What this repository contains

```
.
├── agentroute_jang2005_complete.py   # AgentRoute core: pattern classifier, LRU cache, registry
├── medqa_evaluation.py               # MedQA benchmark + the 6-router comparison library
├── gpqa_evaluation.py                # GPQA Diamond benchmark (same 6 routers)
├── scalability_sweep.py              # Agent-pool sweep: message volume + routing latency vs N
├── production_casestudy.py           # Claude API production case study (rate limits, budget cap)
├── concurrent_load_test.py           # Async QPS load test: p50/p95/p99 tail latency
├── requirements.txt
└── paper/                            # Paper draft (LaTeX) and bibliography
    ├── paper.tex
    └── references.bib
```

`medqa_evaluation.py` is the shared library: it defines the six router
implementations and the MCQ helpers that the other scripts import. Keep all
Python files in the same directory — the scripts import each other by module
name from the repository root.

## Routers compared

All six share a single LLM backbone per experiment, so only routing behaviour
differs:

| Router | Description |
|---|---|
| **AgentRoute** | Deterministic pattern routing + LRU cache, one LLM call (this work) |
| OI-MAS | Confidence-aware role/model routing with a verifier |
| RCR-Router | Role-aware context routing (Planner / Searcher / Recommender) |
| EvoMAS | Evolutionary multi-agent configuration search |
| DyTopo | Dynamic communication topology via semantic matching |
| RopMura | Four-submodule planner with cluster-embedding routing |

The baselines are re-implemented to follow each method's published architecture.
References are in `paper/references.bib`.

## Quick start

```bash
pip install -r requirements.txt

# Smoke test (a few minutes on one GPU): confirms the pipeline end-to-end
python medqa_evaluation.py --num_problems 10 --runs 1 --seeds 13
```

A 16 GB GPU is sufficient for the default Qwen 2.5-3B backbone. Each script has a
`HOW TO RUN` block at the bottom with smoke-test and full-run commands.

## Reproducibility

- Every script seeds `random`, `numpy`, and `torch`; seeds are command-line
  arguments and recorded in each output row.
- Statistical analyses (paired t-tests vs AgentRoute, Cohen's d, 95 % CIs,
  Bonferroni correction, Friedman test) are computed inside the scripts.
- Default backbone: `Qwen/Qwen2.5-3B-Instruct` (fp16). The Claude case study
  uses the Anthropic API and requires `ANTHROPIC_API_KEY`.

## Notes

- The Claude case study and the optional Claude backend of the load test require
  an Anthropic API key (`export ANTHROPIC_API_KEY=...`) and incur API cost.
- GPQA Diamond is a gated dataset; downloading it requires accepting the license
  on the Hugging Face Hub and setting `HF_TOKEN`.
